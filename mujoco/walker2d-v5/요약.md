# Walker2D-v5 환경 설명

Walker2D-v5는 MuJoCo 물리 엔진을 기반으로 한 OpenAI Gymnasium의 대표적인 연속 제어 환경입니다. 이 환경은 2D 평면에서 보행하는 양다리 로봇을 시뮬레이션하며, 강화학습 알고리즘의 성능을 테스트하는 데 널리 사용됩니다.

## 환경 개요

Walker2D는 다음과 같은 특징을 가집니다:

1. **로봇 구조**: 7개의 강체(몸통과 다리 파트)로 구성된 2차원 양발 보행 로봇입니다.

2. **자유도**: 총 6개의 회전 관절(각 다리에 3개씩)을 가지고 있으며, 이를 통해 인간과 유사한 보행 동작을 수행할 수 있습니다.

3. **목표**: 로봇이 넘어지지 않고 오른쪽으로 최대한 빠르게 걷도록 학습하는 것입니다.

## 기술적 세부 사항

### 상태 공간 (State Space)
- **관측 차원**: 17차원 벡터
- **포함 정보**: 
  - 각 신체 부위의 위치와 회전 정보
  - 각 관절의 각도와 각속도
  - 선형 속도와 각속도

### 행동 공간 (Action Space)
- **차원**: 6차원 벡터 (6개 관절에 대한 토크 제어)
- **범위**: 각 행동 차원은 [-1, 1] 범위로 정규화됨
- **의미**: 각 관절에 적용되는 토크의 크기

### 보상 함수 (Reward Function)
Walker2D-v5의 보상 함수는 다음 요소들로 구성됩니다:

1. **전진 보상**: x축 방향으로의 이동 속도에 비례하는 보상
2. **에너지 소비 페널티**: 과도한 관절 토크 사용에 대한 작은 페널티
3. **생존 보너스**: 로봇이 넘어지지 않고 계속 서있을 때 받는 작은 상수 보상
4. **정지 페널티**: 로봇이 너무 늦게 움직이거나 멈추면 받는 페널티

### 종료 조건 (Termination)
에피소드는 다음 조건에서 종료됩니다:

1. 로봇의 몸통 높이가 너무 낮아짐 (넘어짐)
2. 로봇의 몸통 기울기가 너무 커짐 (균형 상실)
3. 최대 타임스텝(1000)에 도달

## 학습의 난이도

Walker2D-v5는 다음과 같은 이유로 강화학습에 중간 수준의 난이도를 제공합니다:

1. **높은 자유도**: 6개의 관절을 동시에 제어해야 합니다.
2. **불안정한 역학**: 로봇이 쉽게 넘어질 수 있어 안정적인 보행이 어렵습니다.
3. **신용 할당 문제**: 현재의 행동과 미래의 보상 사이의 지연된 관계를 학습해야 합니다.
4. **탐색의 어려움**: 초기에는 로봇이 대부분 넘어지므로 유용한 전략을 발견하기 어렵습니다.

## Hopper-v5와의 차이점

앞서 보신 코드에서 다뤘던 Hopper-v5와 비교했을 때 주요 차이점은:

1. **복잡성**: Walker2D는 두 다리를 가지고 있어 Hopper(한 다리)보다 더 복잡합니다.
2. **자유도**: Walker2D는 6개의 액추에이터(관절)를 가진 반면, Hopper는 3개만 가집니다.
3. **보행 역학**: Walker2D는 좀 더 인간형 보행을 모방하도록 설계되었습니다.
4. **학습 난이도**: 일반적으로 Walker2D는 Hopper보다 학습하기 더 어렵습니다.

## 알고리즘 적용 시 고려사항

Walker2D-v5 환경에 PPO 같은 알고리즘을 적용할 때 고려해야 할 사항:

1. **더 긴 학습 시간**: Hopper보다 더 많은 에피소드와 시간이 필요합니다.
2. **더 넓은 네트워크**: 복잡한 정책을 표현하기 위해 더 큰 신경망이 필요할 수 있습니다.
3. **세심한 하이퍼파라미터 튜닝**: 특히 탐색-활용 균형과 학습률 조정이 중요합니다.
4. **리워드 쉐이핑**: 초기 학습 단계에서 보조적인 보상 신호를 추가하는 것이 도움될 수 있습니다.

Walker2D-v5는 로봇 제어와 강화학습 알고리즘 개발에 있어 중요한 벤치마크 환경으로, 성공적으로 학습시키면 알고리즘의 견고성을 검증할 수 있습니다.
